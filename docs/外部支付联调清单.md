# Router 外部支付联调清单

## 1. 目标

本文档用于打通以下链路：

1. `router.yeying.pub` 创建业务订单
2. `router` 服务端调用 `www.tidukongjian.com` 外部支付下单 API
3. 跳转到 API 返回的 `cashier_url`
4. 微信支付完成
5. `tidukongjian` 回调 `router`
6. `router` 发放余额或套餐权益

当前默认按方案 B，也就是“支付能力 API 化”模式联调；旧的跳转拼参模式仅保留兼容。

## 2. 联调边界

`router` 负责：

- 业务订单
- 用户余额充值
- 套餐购买
- 回调后的发货
- 订单状态推进

`tidukongjian` 负责：

- 收银台页面
- 微信统一下单
- 支付结果接收
- 支付结果回调 `router`

## 4. 配置项总表

### 4.1 Router 必配

| 配置项 | 位置 | 作用 | 示例值 |
| --- | --- | --- | --- |
| `ServerAddress` | `router` 配置文件 `config.yaml` | 生成 `callback_url` 基础域名 | `https://router.yeying.pub` |
| `TopUpMode` | `router` 配置文件 `config.yaml` | 支付接入模式 | `api` |
| `TopUpAPICreateURL` | `router` 配置文件 `config.yaml` | 外部支付下单 API 地址 | `https://www.tidukongjian.com/addons/ymq_zixishi/public/index.php/addons/ymqzixishi/api.external_pay/create` |
| `TopUpAPIQueryURL` | `router` 配置文件 `config.yaml` | 外部支付查单 API 地址，留空时可从 create URL 自动推导 | `https://www.tidukongjian.com/addons/ymq_zixishi/public/index.php/addons/ymqzixishi/api.external_pay/query` |
| `TopUpAPIUniacid` | `router` 配置文件 `config.yaml` | 外部支付接口 `uniacid` | `1` |
| `TopUpMerchantApp` | `router` 配置文件 `config.yaml` | 外部商户标识 | `router` |
| `TopUpSignSecret` | `router` 配置文件 `config.yaml` | 生成支付请求签名 | `router_pay_sign_202604` |
| `TopUpCallbackToken` | `router` 配置文件 `config.yaml` | 校验 `tidukongjian` 回调 | `router_pay_callback_202604` |

说明：

- `TopUpSignSecret` 和 `TopUpCallbackToken` 必须与 `tidukongjian` 对应商户配置保持一致。
- `ServerAddress` 不能填内网地址，否则 `tidukongjian` 无法回调 `router`。

### 4.2 Tidu 必配

| 配置项 | 位置 | 作用 | 示例值 |
| --- | --- | --- | --- |
| `wx_pay_app_id` | 插件配置 | 微信支付应用 ID | 按实际商户填写 |
| `wx_pay_mch_id` | 插件配置 | 微信商户号 | 按实际商户填写 |
| `wx_pay_key` | 插件配置 | 微信支付 API Key | 按实际商户填写 |
| `wx_pay_cert_path` | 插件配置 | 微信证书路径 | `/uploads/wxpay/apiclient_cert.pem` |
| `wx_pay_key_path` | 插件配置 | 微信私钥路径 | `/uploads/wxpay/apiclient_key.pem` |
| `external_pay_merchants` | 插件配置 | 外部商户配置 JSON | 见下方示例 |

`external_pay_merchants` 推荐示例：

```json
{
  "router": {
    "sign_secret": "router_pay_sign_202604",
    "callback_token": "router_pay_callback_202604",
    "callback_url": "https://router.yeying.pub/api/v1/public/topup/callback"
  }
}
```

说明：

- `merchant_app` 当前固定使用 `router`。
- `callback_url` 可由 `router` 请求里动态带入，但建议配置默认值，方便排障和兜底。
- 生产、测试环境不要共用同一组密钥和回调令牌。

## 5. 请求与回调地址

### 5.1 Router 发起到 Tidu

下单 API 地址：

```text
POST https://www.tidukongjian.com/addons/ymq_zixishi/public/index.php/addons/ymqzixishi/api.external_pay/create?uniacid=1
```

Router 创建订单接口：

- `POST /api/v1/public/user/topup/orders`

两类业务：

- `balance_topup`
- `package_purchase`

### 5.2 Tidu 回调到 Router

回调接口：

```text
POST https://router.yeying.pub/api/v1/public/topup/callback
```

鉴权头：

```text
X-Topup-Callback-Token: router_pay_callback_202604
```

## 7. 联调前检查清单

### 7.1 Router 侧

- `ServerAddress` 已配置为公网 HTTPS 域名
- `TopUpMode` 已配置为 `api`
- `TopUpAPICreateURL` 已配置为 `tidukongjian` 外部支付下单地址
- `TopUpSignSecret` 已配置
- `TopUpCallbackToken` 已配置
- 至少存在 1 个启用套餐，并且已配置：
  - `sale_price`
  - `sale_currency`
- `billing_currencies` 中已存在 `CNY`
- `CNY -> YYC` 汇率已确认

### 7.2 Tidu 侧

- `wx_pay_app_id` 已配置
- `wx_pay_mch_id` 已配置
- `wx_pay_key` 已配置
- 证书路径存在且 PHP 进程可读：
  - `wx_pay_cert_path`
  - `wx_pay_key_path`
- `external_pay_merchants` 已配置 `router`
- `router.sign_secret` 与 `router.TopUpSignSecret` 一致
- `router.callback_token` 与 `router.TopUpCallbackToken` 一致
- `router.callback_url` 已配置为 `router` 公网地址
- 服务器可从 `tidukongjian` 访问 `router.yeying.pub`

### 7.3 网络与环境

- `router.yeying.pub` 可以被公网访问
- `www.tidukongjian.com` 可以被公网访问
- 两边服务器时间基本一致
- HTTPS 证书正常
- 防火墙未拦截回调请求

## 8. 联调顺序

### 第 1 步：先校验配置

先人工核对：

1. `router` 的支付配置是否已填
2. `tidukongjian` 的 `external_pay_merchants.router` 是否已填
3. 双方共享密钥和回调令牌是否一致

### 第 2 步：校验套餐购买链路

建议先测套餐购买，因为金额固定，排查更直接。

1. 在 `router` 后台新增或确认一个已启用套餐
2. 套餐已配置 `sale_price` 和 `sale_currency`
3. 用户打开 `router` 充值页
4. 调用 `GET /api/v1/public/user/packages` 确认前端能拿到套餐
5. 点击购买套餐
6. 调用 `POST /api/v1/public/user/topup/orders`
7. `router` 服务端调用 `api.external_pay/create`
8. 返回 `redirect_url`，其值为 `cashier_url`
9. 跳转到 `tidukongjian` 收银台
10. 收银台成功拉起微信支付
11. 支付完成后 `tidukongjian` 回调 `router`
12. `router` 订单推进到 `fulfilled`
13. 用户套餐权益到账

### 第 3 步：校验余额充值链路

1. 用户输入人民币金额
2. 调用 `POST /api/v1/public/user/topup/orders`
3. 请求体中：
   - `business_type=balance_topup`
   - `amount=<CNY 金额>`
4. 后端按 `CNY -> YYC` 汇率计算到账额度
5. 跳转支付
6. 支付完成
7. `router` 记录充值日志
8. 用户余额增加

### 第 4 步：校验失败分支

至少验证以下失败场景：

1. `TopUpSignSecret` 不一致时，收银台应报“支付签名校验失败”
2. `TopUpCallbackToken` 不一致时，`router` 回调接口应返回未授权
3. 套餐未配置售价时，`router` 下单接口应直接失败
4. `callback_url` 不可访问时，`tidukongjian` 应记录 `callback_status=失败`
5. 支付金额不一致时，`tidukongjian` 应拒绝入账

## 9. 建议的实际联调顺序

建议严格按下面顺序，不要一开始就上真实支付：

1. 配好双方配置
2. 打开 `router` 充值页，确认套餐列表可见
3. 用浏览器直接打开收银台地址，确认页面可达
4. 调用 `POST /api/v1/public/user/topup/orders`，确认能拿到 `redirect_url`
5. 打开 `redirect_url`，确认 `tidukongjian` 能创建外部支付单
6. 再做一笔最小金额真实支付
7. 支付后检查：
   - `tidukongjian` 外部支付单状态
   - `router` 订单状态
   - `router` 发货结果

## 10. 排障重点

### 10.1 打开收银台直接报错

优先检查：

- `merchant_app`
- `order_id`
- `transaction_id`
- `amount`
- `callback_url`
- `sign`

### 10.2 收银台能打开，但拉不起微信支付

优先检查：

- `wx_pay_app_id`
- `wx_pay_mch_id`
- `wx_pay_key`
- 证书路径
- 微信商户配置是否支持当前支付方式

### 10.3 支付成功了，但 Router 没发货

优先检查：

- `external_pay_merchants.router.callback_token`
- `external_pay_merchants.router.callback_url`
- `router.ServerAddress`
- `POST /api/v1/public/topup/callback` 返回内容

### 10.4 余额到账不对

优先检查：

- `billing_currencies` 中 `CNY` 的 `yyc_per_unit`
- 订单创建时传入的 `amount`
- `router` 计算出的 `quota`

### 10.5 套餐购买失败

优先检查：

- 套餐是否启用
- 套餐是否配置 `sale_price`
- 套餐是否配置 `sale_currency`

## 11. 联调完成标准

以下全部满足，才算联调完成：

1. 套餐购买成功 1 次
2. 余额充值成功 1 次
3. 两条链路都完成真实支付
4. 两条链路都完成回调
5. 两条链路都完成发货
6. 失败场景至少验证 2 项
