# 模型端点

本文档只作为模型、端点、渠道、分组这组规则的总览入口。

当前不再在这一页重复定义具体规则，避免和专题文档出现多份口径。

## 1. 文档分工

### 1.1 供应商层

见 [供应商规范](./供应商规范.md)。

负责内容：

1. 供应商本身。
2. 供应商模型。
3. 模型官方价格。
4. 模型官方端点范围。
5. 模型状态、类型、归属等官方信息。

适合查看的问题：

1. 某个模型的官方价格从哪里来。
2. 某个模型官方支持哪些端点。
3. 供应商新增模型或模型下线时，migration 应该怎么写。

### 1.2 渠道层

见 [渠道规范](./渠道规范.md)。

负责内容：

1. 渠道接入哪些模型。
2. 渠道模型是否启用。
3. 渠道模型映射到哪个上游模型。
4. 渠道端点是否启用。
5. 最近一次上游返回结果和最近一次测试结果如何记录。

适合查看的问题：

1. 人工配置真值在哪一层。
2. refresh 能改什么，不能改什么。
3. 测试结果、上游返回结果和人工配置是什么关系。
4. 为什么某个模型或端点当前不能启用。

### 1.3 分组层

见 [分组规范](./分组规范.md)。

负责内容：

1. 分组暴露哪些模型。
2. 分组接入哪些渠道。
3. 分组内渠道优先级。
4. `group_model_channels` 这类静态展开关系。
5. `/v1/models` 和 `supported_endpoints` 的分组输出边界。

适合查看的问题：

1. 为什么某个模型在某分组下可见或不可见。
2. `group_channels` 和 `group_model_channels` 分别是什么。
3. 分组保存后为什么需要刷新静态展开关系。

### 1.4 端点协议层

见 [端点协议](./端点协议.md)。

负责内容：

1. Router 当前支持哪些端点协议。
2. `/v1/chat/completions`、`/v1/responses`、`/v1/messages` 的差异。
3. 不同供应商或渠道通常暴露哪种协议。
4. 管理端新增或编辑渠道时应如何选择协议类型。

适合查看的问题：

1. Claude 渠道为什么不一定要走 `/v1/messages`。
2. OpenAI 兼容渠道为什么应该选 `openai` 协议。
3. 当前文本请求为什么要求同端点转发。

### 1.5 接口与运行时

接口字段、接口路径和返回结构，见 [接口文档](./接口文档.md)。

实际 Relay 选路、请求内切换、端点校验与转发流程，见 [路由逻辑](./路由逻辑.md)。

## 2. 最小阅读路径

如果只是排查问题，建议按下面顺序读：

1. 先看 [接口文档](./接口文档.md)，确认接口和字段。
2. 再看 [路由逻辑](./路由逻辑.md)，确认请求实际怎么走。
3. 看到供应商问题时看 [供应商规范](./供应商规范.md)。
4. 看到渠道模型、端点、测试、refresh 问题时看 [渠道规范](./渠道规范.md)。
5. 看到分组暴露、分组绑定、`/v1/models` 输出问题时看 [分组规范](./分组规范.md)。
6. 看到 `/v1/chat/completions`、`/v1/responses`、`/v1/messages` 差异时看 [端点协议](./端点协议.md)。

## 3. 当前总原则

这里只保留最上层的总原则：

1. 官方信息、人工配置、上游返回结果、测试结果、静态展开关系，必须分开理解。
2. 供应商层不替渠道层做启停决策。
3. 渠道层不因 refresh 或测试静默覆盖人工配置。
4. 分组层不拥有渠道端点真值。
5. `/v1/models` 和 `supported_endpoints` 只能暴露经过渠道层和分组层处理后的结果。

如果需要具体规则，不在本页展开，直接进入对应专题文档。
