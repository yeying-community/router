# 国产 API 渠道微测报告（2026-05-14）

## 1. 报告范围

本报告聚焦 4 个已接通的国产 API 渠道，并统一使用各自当前代表性文本模型做一轮**低成本、零调参、OpenAI 兼容接入**微测：

- 智谱 `glm-5.1`
- 阿里云百炼 `qwen3.6-max-preview`
- DeepSeek `deepseek-v4-pro`
- 火山方舟 `doubao-seed-2-0-pro-260215`

这份报告回答 3 个问题：

1. `2026-05-14` 当天这些模型是否能直接调通。
2. 在**默认 OpenAI-compatible 接入**下，它们的“严格式输出、时延、成本”是什么感觉。
3. 结合官方文档，它们在多模态、Realtime 语音、OpenAI/Anthropic 兼容性上的覆盖面如何。

## 2. 一页结论

- 如果你要**默认接入就尽量少踩坑**，本轮微测里 `DashScope` 和 `火山方舟` 的严格式输出最好。
- 如果你要**最省钱、最快**，本轮微测里 `DeepSeek-v4-pro` 最突出。
- 如果你要**综合能力面最完整**，文档面上 `DashScope` 仍然最强。
- 如果你要**文本 / Coding 主榜候选**，`GLM-5.1` 值得进正式 benchmark，但它在本轮“零调参严格式输出”上不占优。
- 如果你要**图像 / 视频 / Realtime 语音**，`DashScope`、`火山方舟`、`智谱` 都值得单独拉副榜；`DeepSeek` 不适合和它们混比多模态。

## 3. 雷达图风格总览

## 4. 测试设计

### 4.1 微测原则

- 统一走 `chat/completions`
- 统一使用 OpenAI 兼容入口
- 尽量少加供应商私有参数
- 使用 `temperature=0`
- 控制 `max_tokens`，把成本压到极低

### 4.2 两个测试题

| Case | 类型 | 目标 |
| --- | --- | --- |
| `L1` | 逻辑 + 严格式输出 | 要求模型**只输出 JSON 数组**，不允许解释 |
| `J1` | 信息抽取 + 严格式 JSON | 要求模型**只输出 JSON**，不允许解释和 markdown |

### 4.3 本轮最重要的口径

本轮故意比较的是：

- **“默认 OpenAI-compatible 接入体验”**
- 不是“针对每家模型深度调参后谁最强”

这会天然放大“谁更爱解释、谁更愿意直接给结果”的差异。  
对真实生产很有参考价值，因为很多项目在第一版接入时就是这样接的。

## 6. 怎么解读这轮结果

### 6.1 如果你看“默认接入就能不能用”

- `DashScope` 最稳，几乎就是“花钱换省心”。
- `火山方舟` 在本轮最像“均衡解”，没有 `DashScope` 那么贵，也没有 `GLM / DeepSeek` 那种默认就想展开讲的倾向。

### 6.2 如果你看“性能价格比”

- `DeepSeek-v4-pro` 本轮明显最便宜，且速度也快。
- 问题不是能力不够，而是默认参数下对“只要结果不要过程”的服从性不如 `DashScope / 火山`。

### 6.3 如果你看“文本/Coding 天花板”

- 本轮微测**不能据此判定** `GLM-5.1` 弱。
- 这轮只能说明：在“OpenAI 兼容 + 极简参数 + 严格式结果”这个窄场景里，`GLM-5.1` 的默认输出风格不占便宜。
- 如果你后续要做正式主榜，建议再给 `GLM-5.1`、`DeepSeek-v4-pro` 加一轮“关闭/降低思考、加强 JSON 约束”的正式复测。

## 7. 雷达图打分方法

## 8. 复现代码

### 8.1 最简运行方式

完整脚本：[`reproduce_microbench_ascii.py`](./reproduce_microbench_ascii.py)

PowerShell 示例：

```powershell
$env:ZHIPU_API_KEY="your-zhipu-key"
$env:DASHSCOPE_API_KEY="your-dashscope-key"
$env:DEEPSEEK_API_KEY="your-deepseek-key"
$env:ARK_API_KEY="your-ark-key"

python .\reproduce_microbench_ascii.py
```

脚本会输出一个 `microbench_ascii_latest.json`，包含：

- 每次调用的 `http / latency / tokens / strict_hit / answer_preview`
- 每个供应商的汇总
- 本报告使用的价格假设

### 8.2 复现脚本核心代码

```python
import json
import os
import time
import urllib.error
import urllib.request

VENDORS = [
    {
        "vendor": "Zhipu",
        "model": "glm-5.1",
        "url": "https://open.bigmodel.cn/api/paas/v4/chat/completions",
        "env": "ZHIPU_API_KEY",
    },
    {
        "vendor": "DashScope",
        "model": "qwen3.6-max-preview",
        "url": "https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions",
        "env": "DASHSCOPE_API_KEY",
    },
    {
        "vendor": "DeepSeek",
        "model": "deepseek-v4-pro",
        "url": "https://api.deepseek.com/chat/completions",
        "env": "DEEPSEEK_API_KEY",
    },
    {
        "vendor": "Ark",
        "model": "doubao-seed-2-0-pro-260215",
        "url": "https://ark.cn-beijing.volces.com/api/v3/chat/completions",
        "env": "ARK_API_KEY",
    },
]

CASES = [
    {
        "id": "L1",
        "prompt": (
            "Return only the final answer as a JSON array with no explanation. "
            "Choose 3 items from A,B,C,D,E with these constraints: "
            "B must be included; A and C cannot both be included; "
            "if D is included then E cannot be included. "
            "Choose the lexicographically smallest valid set."
        ),
        "strict_check": lambda s: s.replace(" ", "").strip() == '["A","B","D"]',
        "max_tokens": 120,
    },
    {
        "id": "J1",
        "prompt": (
            "Return only JSON with no explanation and no markdown. "
            "Text: Order O-2048. Customer: Hangzhou Yunqi Tech Co., Ltd. "
            "Contact: Li Min. Phone: 13800138000. "
            "Items: GPU Server, qty 3, unit price 42000; "
            "Switch, qty 2, unit price 6800. "
            "Deliver to Pudong, Shanghai before 2026-05-20. "
            "Required fields: order_id, customer, contact, phone, items, "
            "delivery_deadline, delivery_city, delivery_district, total_amount."
        ),
        "strict_check": lambda s: (
            s.lstrip().startswith("{")
            and '"order_id"' in s
            and '"customer"' in s
            and "O-2048" in s
            and "139600" in s
            and "Pudong" in s
            and "Shanghai" in s
        ),
        "max_tokens": 220,
    },
]

def post_json(url, api_key, payload):
    body = json.dumps(payload, ensure_ascii=True).encode("utf-8")
    req = urllib.request.Request(url, data=body, method="POST")
    req.add_header("Authorization", f"Bearer {api_key}")
    req.add_header("Content-Type", "application/json")
    start = time.time()
    try:
        with urllib.request.urlopen(req, timeout=180) as resp:
            raw = resp.read().decode("utf-8", errors="ignore")
            return resp.status, json.loads(raw), raw, round((time.time() - start) * 1000)
    except urllib.error.HTTPError as exc:
        raw = exc.read().decode("utf-8", errors="ignore")
        return exc.code, json.loads(raw), raw, round((time.time() - start) * 1000)

for case in CASES:
    for spec in VENDORS:
        payload = {
            "model": spec["model"],
            "messages": [{"role": "user", "content": case["prompt"]}],
            "max_tokens": case["max_tokens"],
            "temperature": 0,
        }
        status, data, raw, latency_ms = post_json(
            spec["url"], os.environ[spec["env"]], payload
        )
        message = data["choices"][0].get("message", {}) if data.get("choices") else {}
        content = message.get("content", "") or message.get("reasoning_content", "") or ""
        print(case["id"], spec["vendor"], status, latency_ms, case["strict_check"](content))
```

## 9. 价格估算口径

> 这部分只用于**粗估本轮 8 次调用花了多少钱**，不是长期采购价承诺。  
> 不同供应商的模型会按输入长度、输出长度、地域、活动期、缓存命中与否产生差异。

本报告使用的估算价：

- 智谱 `GLM-5.1`：按 `<32K` 档 `6 / 24 元/百万 Token`
- 百炼 `qwen3.6-max-preview`：按当前公开高阶推理档 `9 / 54 元/百万 Token`
- DeepSeek `deepseek-v4-pro`：按公开页 `3 / 6 元/百万 Token`
- 火山方舟 `Doubao-Seed-2.0-pro`：按公开可检索价位 `6.4 / 32 元/百万 Token`

本轮 8 次调用粗估成本：

- 智谱：约 `¥0.0093`
- 百炼：约 `¥0.2912`
- DeepSeek：约 `¥0.0026`
- 火山方舟：约 `¥0.0652`
- 合计：约 `¥0.3682`

## 11. 报告限制

- 这轮是**低成本微测**，只有 2 道题，不足以代表完整能力上限。
- 它更适合回答“默认接入体验、严格式输出、速度、成本”。
- 真要采购拍板，建议再跑一轮正式小榜：
- 中文复杂推理
- 严格 JSON / Function Call
- 长程 Coding / 真实仓库修复
