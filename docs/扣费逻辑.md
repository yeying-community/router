# 扣费逻辑

本文档是 Router 当前版本“额度使用 + 金额语义 + 请求期扣减”的统一说明。

边界：

- 支付下单、回调、订单状态、充值履约：见 [支付方案（定稿）](./支付方案.md)
- 模型单价解析与复杂计费条件：见 [模型计费](./模型计费.md)

## 1. 核心基线

1. 平台内部统一记账单位是 `夜莺币`，简称 `YYC`（Yeying Coin）。
2. 上游模型价格按官方币种维护（如 `USD`、`CNY`）。
3. 请求结算时，先算原币种金额，再折算为 `YYC` 入账。
4. 旧字段名 `quota` 系列继续保留兼容，但业务语义统一解释为 `YYC`。

## 2. 术语与字段语义

### 2.1 术语

| 术语 | 含义 |
| --- | --- |
| `YYC` | 平台内部统一记账单位 |
| `amount` | 面向展示或支付的法币金额 |
| `price` | 上游供应商单价 |
| `currency` | 价格/金额所属币种 |

### 2.2 兼容字段与显式字段（当前口径）

| 兼容字段 | 显式字段 | 统一语义 | 典型返回接口 |
| --- | --- | --- | --- |
| `quota` | `yyc_balance` / `yyc_amount` / `yyc_value` | `YYC` | `quota`、`yyc_balance`（用户/列表）；`yyc_amount`、`yyc_value`（按各业务接口） |
| `used_quota` | `yyc_used` | 已消费 `YYC` | `used_quota`、`yyc_used`（用户/令牌/日志等） |
| `remain_quota` | `yyc_remain` | 剩余 `YYC` | 令牌相关接口 |
| `daily_quota_limit` | 无 | 每日额度（`YYC`） | 套餐/订阅相关接口 |
| `package_emergency_quota_limit` | 无 | 应急额度（`YYC`） | 套餐/订阅相关接口 |
| `consumed_quota` | `yyc_consumed` | 已结算 `YYC` | 额度快照接口 |
| `reserved_quota` | `yyc_reserved` | 已预占 `YYC` | 额度快照接口 |
| `remaining_quota` | `yyc_remaining` | 剩余可用 `YYC` | 额度快照接口 |
| `user_daily_quota` | `yyc_user_daily` | 本次由套餐日额度承担的 `YYC` | 消费日志接口 |
| `user_emergency_quota` | `yyc_user_emergency` | 本次由套餐应急额度承担的 `YYC` | 消费日志接口 |

规则：

1. 显式 `yyc_*` 字段仅在已标注接口返回；不假设全量接口统一返回。
2. 兼容字段继续保留，且数值必须与显式字段一致。
3. `amount` 只表示法币，不表示 `YYC`。

### 2.3 金额快照字段

| 字段 | 语义 | 单位 |
| --- | --- | --- |
| `billing_input_amount` | 输入侧法币金额快照 | `billing_currency` |
| `billing_output_amount` | 输出侧法币金额快照 | `billing_currency` |
| `billing_amount` | 本次法币总金额快照 | `billing_currency` |
| `billing_yyc_rate` | 计费时汇率快照 | `YYC / currency` |
| `billing_yyc_amount` | 本次最终 `YYC` | `YYC` |

## 3. 价格折算与记账

### 3.1 折算流程

```text
上游价格（price + currency + unit）
  -> 按请求用量计算法币金额
  -> 按币种汇率折算为 YYC
  -> 按 YYC 扣减余额 / 套餐额度 / 令牌额度
```

### 3.2 统一公式（按当前代码）

```text
raw_yyc = amount(currency) × yyc_per_unit(currency) × group_ratio
yyc_cost = ceil(raw_yyc)
```

说明：

1. `amount(currency)` 由 `price_unit` 决定：
- `per_1k_tokens` / `per_1k_chars`：`amount = quantity × price / 1000`
- `per_image` / `per_video` / `per_second` / `per_minute` / `per_request` / `per_task`：`amount = quantity × price`
2. `group_ratio` 来自分组业务倍率。
3. `yyc_cost` 最终按 `YYC` 整数向上取整。
4. 当本次有用量但价格换算后为 0（且非免费分组）时，最小记 1 `YYC`。
5. 当 `group_ratio=0`（免费分组）时，记账结果为 0 `YYC`。

## 4. 请求期扣减模型

### 4.1 两种扣费来源

| 模式 | 触发条件 | 扣减对象 | `billing_source` |
| --- | --- | --- | --- |
| 套餐模式 | 当前分组有生效套餐，且套餐日额度+套餐应急额度可覆盖预占 | `group_quota_counters` + `user_quota_counters(package_emergency)` | `package` |
| 余额模式 | 无生效套餐，或套餐额度不足回落 | `users.quota`（并同步余额批次交易） | `balance` |

说明：

- 内部存在 `package_fallback_balance` 场景；日志对外仍归因为 `balance`。
- 两种模式下若配置令牌限制，令牌额度会同步扣减。

### 4.2 准入与预占

1. 先计算本次预估 `YYC`。
2. 判断当前分组是否存在生效套餐。
3. 若有套餐：尝试 `ReservePackageQuota`。
4. 套餐预占成功则走套餐模式。
5. 套餐预占因额度不足失败则自动回落余额模式。
6. 余额模式下检查主余额与令牌额度。

### 4.3 结算与回滚

1. 请求成功：
- 套餐模式先结算日额度，再结算套餐应急额度。
- 余额模式扣减 `users.quota` 并写余额交易。
2. 请求失败：
- 释放预占，并归还预扣余额/令牌。

### 4.4 预估与实扣差异（当前实现）

1. 准入阶段按“预估 `YYC`”预占（避免并发超扣）。
2. 结算阶段按真实 usage 重算 `YYC`，并对“预估 - 实际”做差额回补或补扣。
3. 余额模式在“主余额远大于预估值”（`user_quota > 100 × preConsumedQuota`）时，会跳过数据库层预扣，最终按实耗落账。
4. 套餐模式只要预占成功，就始终先在套餐计数器结算；套餐不足才会在准入阶段回落到余额模式。

## 5. 周期与计数器口径

1. 分组套餐日额度：`group_quota_counters(counter_type=daily)`，按自然日。
2. 套餐应急额度：`user_quota_counters(counter_type=package_emergency)`，按自然月。
3. 周期边界时区：`quota_reset_timezone`，空值回落 `Asia/Shanghai`。
4. 分组日额度策略来自“当前分组生效套餐”；无生效套餐时该分组视图口径为 unlimited（`limit=0`）。

## 6. 余额批次口径

余额模式下，批次扣减采用 FEFO（先到期先用），不是 FIFO（最早充值先用）。

排序规则：

1. `expires_at=0`（永久有效）排最后。
2. 其余按 `expires_at asc`。
3. 同到期时间按 `created_at asc`，再 `id asc`。

## 7. 对外接口口径

### 7.1 快照接口

- `GET /api/v1/public/user/quota/daily`
- `GET /api/v1/public/user/quota/summary`
- `GET /api/v1/admin/user/:id/quota/summary`
- `GET /api/v1/admin/group/:id/quota/daily?user_id=...`

快照对象统一提供：

- `limit / yyc_limit`
- `consumed_quota / yyc_consumed`
- `reserved_quota / yyc_reserved`
- `remaining_quota / yyc_remaining`

### 7.2 已移除旧快照列

1. `users.daily_quota_limit`、`users.package_emergency_quota_limit` 已从用户表迁移移除。
2. 运行时额度策略统一来自“生效套餐 + 额度快照计数器”，不从用户表读取额度上限。

## 8. 页面与文案规范

1. 页面术语统一使用 `YYC`，不再用 `quota` 作为产品文案。
2. 若同时展示法币，采用双值展示：`1250 YYC（约 $0.0025）`。
3. 兑换比率建议使用“`USD -> YYC`”表达，不使用“单位美元额度”。

## 9. 当前边界（代码现状）

1. 日志 `billing_source` 当前只落两类：`package`、`balance`（不落 `package_fallback_balance`）。
2. 套餐准入基于 `ReservePackageQuota`（分组日额度 + 套餐应急额度），余额准入基于主余额与令牌额度。
3. 计价与记账保留双快照：法币金额（`billing_*_amount`）与 `YYC`（`billing_yyc_amount`）。
