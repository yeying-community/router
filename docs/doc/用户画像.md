  Chat 用户画像（面向最终使用聊天的用户）

  1. 轻量问答型

  - 目标：快速得到答案/总结/翻译
  - 关心：速度、准确性、成本可控
  - 痛点：答非所问、响应慢、费用不透明
  - 成功指标：一次到位、少改动、能省时间

  2. 产出协作型

  - 目标：写作/代码/文案/方案的共创
  - 关心：上下文连续、可追溯、可复制
  - 痛点：上下文丢失、格式不统一、需要反复提示
  - 成功指标：能产出可直接用的初稿

  3. 专业场景型

  - 目标：在特定领域工作流中提高效率
  - 关心：稳定、可控、与业务数据或工具集成
  - 痛点：权限、合规、与现有工具割裂
  - 成功指标：节省流程步骤、少出错

  Router 用户画像（面向路由/后台/运维管理的用户）

  1. 业务运营/产品侧

  - 目标：控制成本、优化体验、看清使用趋势
  - 关心：花费、token、请求次数、模型质量
  - 痛点：数据看不懂、口径不统一、定位问题慢
  - 成功指标：费用可预测、体验稳定

  2. 后端/平台工程师

  - 目标：稳定接入、多模型调度、保障可用性
  - 关心：鉴权、路由策略、限流、监控与告警
  - 痛点：配置复杂、文档缺失、排错慢
  - 成功指标：高可用、低故障、切换顺滑

  3. 运维/安全

  - 目标：可审计、合规、快速恢复
  - 关心：日志、权限、密钥、回滚与备份
  - 痛点：缺少标准流程、监控不闭环
  - 成功指标：出问题可快速定位并恢复
